An agent is a model plus a **harness**. The model reasons; the harness is the infrastructure around it that gives the model the ability to act, remember, and coordinate:

- **Tools**: functions, APIs, and databases the model can call
- **Execution environment**: a filesystem or sandbox where the agent can read, write, and run code
- **Context management**: memory, skills, and summarization that keep the agent effective across long sessions
- **Delegation**: subagents that handle isolated subtasks in their own context windows
- **Steering**: human-in-the-loop approval at critical decision points

[`create_agent`][langchain.agents.create_agent] provides the minimal harness: a model and tools in a loop, with a middleware system that lets you add exactly the capabilities you need. Every item above is opt-in—add what your use case requires, leave out the rest.

The table below maps every built-in capability to its harness category.

| Category | What it covers |
|---|---|
| **[Execution environment](#execution-environment)** | Custom tools, virtual filesystem, code execution |
| **[Context management](#context-management)** | Summarization, memory, skills, prompt caching |
| **[Planning & delegation](#planning-and-delegation)** | Task tracking, subagents |
| **[Fault tolerance](#fault-tolerance)** | Retries, fallbacks, call limits |
| **[Guardrails](#guardrails)** | PII detection and redaction |
| **[Steering](#steering)** | Human-in-the-loop approval |

For the full middleware reference with configuration examples, see [Prebuilt middleware](/oss/javascript/langchain/middleware/built-in).

## Execution environment

The execution environment is how an agent acts in the world. It starts with the tools you wire up—custom functions, APIs, databases—and extends with middleware that adds a managed filesystem and code execution.

| Capability | How to add |
|---|---|
| Custom tools (functions, APIs) | `tools=` parameter on `create_agent` |
| Virtual filesystem + file tools | [`FilesystemMiddleware`](https://reference.langchain.com/javascript/deepagents/middleware/createFilesystemMiddleware) |
| JavaScript REPL (QuickJS) | [`CodeInterpreterMiddleware`](/oss/javascript/deepagents/interpreters) (`langchain_quickjs`) |
| Persistent shell across turns | @[`ShellToolMiddleware`] |
| Isolated sandbox with shell execution | [`SandboxBackend`](/oss/javascript/deepagents/sandboxes) |

## Context management

Context management controls what the agent knows, how long it can run within token limits, and what it retains across sessions.

| Capability | How to add |
|---|---|
| History compression + large-result offloading | [`SummarizationMiddleware`](https://reference.langchain.com/javascript/langchain/index/summarizationMiddleware) |
| Agent-triggered summarization | [`SummarizationToolMiddleware`](https://reference.langchain.com/javascript/deepagents/middleware/createSummarizationMiddleware) |
| Persistent instructions via `AGENTS.md` | [`MemoryMiddleware`](https://reference.langchain.com/javascript/deepagents/middleware/createMemoryMiddleware) |
| On-demand domain knowledge | [`SkillsMiddleware`](https://reference.langchain.com/javascript/deepagents/middleware/createSkillsMiddleware) |
| Prompt caching (Anthropic) | [`AnthropicPromptCachingMiddleware`](https://reference.langchain.com/javascript/langchain/index/anthropicPromptCachingMiddleware) |
| Dynamic tool filtering | @[`LLMToolSelectorMiddleware`] |

## Planning and delegation

Delegation lets agents break large problems into parallel or sequential units of work with isolated context windows.

| Capability | How to add |
|---|---|
| Todo list tracking across turns | [`TodoListMiddleware`](https://reference.langchain.com/javascript/langchain/index/todoListMiddleware) |
| Sync subagent spawning | [`SubAgentMiddleware`](https://reference.langchain.com/javascript/deepagents/middleware/createSubAgentMiddleware) |
| Background non-blocking subagents | [`AsyncSubAgentMiddleware`](https://reference.langchain.com/javascript/deepagents/agent/createDeepAgent) |

## Fault tolerance

Fault tolerance keeps agents running when models or tools fail transiently.

| Capability | How to add |
|---|---|
| Retry on model call failure | [`ModelRetryMiddleware`](https://reference.langchain.com/javascript/langchain/index/modelRetryMiddleware) |
| Retry failed tool calls | [`ToolRetryMiddleware`](https://reference.langchain.com/javascript/langchain/index/toolRetryMiddleware) |
| Fall back to an alternate model | [`ModelFallbackMiddleware`](https://reference.langchain.com/javascript/langchain/index/modelFallbackMiddleware) |
| Cap total model calls per session | [`ModelCallLimitMiddleware`](https://reference.langchain.com/javascript/langchain/index/modelCallLimitMiddleware) |
| Cap total tool calls per session | [`ToolCallLimitMiddleware`](https://reference.langchain.com/javascript/langchain/index/toolCallLimitMiddleware) |

## Guardrails
