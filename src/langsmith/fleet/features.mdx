This page covers the smaller features of LangSmith Fleet that you might miss. For the foundational concepts (channels, sub-agents, memory, approvals), see [Essentials](/langsmith/fleet/essentials).

## Generate and render SVG diagrams

Ask any Fleet agent to draw something and it will write a `.svg` file directly into the agent workspace. The chat renders SVGs inline and shows a thumbnail card you can click to open the file fullscreen.

This is powered by a built-in `svg-builder` skill that ships with every agent. It guides the model to write hand-authored SVG markup (no external rendering scripts) for illustrations, technical diagrams, flowcharts, architecture diagrams, and icons.

<Prompt description="Ask the agent for a diagram and it produces an inline-rendered SVG.">
Draw an SVG diagram of a typical CI/CD pipeline.
</Prompt>

## Generate and render Mermaid diagrams

Fleet agents can produce Mermaid diagrams the same way: by writing a `.mmd` or `.mermaid` file. The chat renders the diagram inline and the file appears in the artifacts sidebar.

The built-in `mermaid-diagram-builder` skill restricts output to the diagram types the renderer supports: `flowchart`, `pie`, `sequence`, `class`, and `state`.

<Prompt description="Ask for a flowchart and the agent renders it inline.">
Make a Mermaid flowchart showing the lifecycle of a support ticket.
</Prompt>

## Generate presentations and present them fullscreen

Fleet ships with a `presentation-builder` skill that writes single-file HTML decks using `data-slide` attributes. The chat detects these decks and shows a slide-aware viewer with:

- Slide-by-slide navigation (arrow keys, on-screen controls)
- Fullscreen presentation mode
- A thumbnail carousel
- A slide counter

<Prompt description="Ask for a slideshow to get a presentable deck.">
Create a 5-slide presentation on the benefits of using a no-code agent platform.

## Mention skills with `/`

Type `/` in the chat input to open a menu of the agent's available [skills](/langsmith/fleet/skills). Pick one to direct the agent to that skill for the next message. This is faster than describing the workflow in prose when you already know which skill applies.

## Mention files with `@`

Type `@` to open a file picker scoped to the agent's workspace. Selecting a file inserts a chip referencing it, so the agent reads that exact file when handling your message. You can also drag a file from the artifacts sidebar into the input to insert a mention.

## Customize the agent icon

Click the agent's icon at the top of the editor to open the icon picker. You can pick from the built-in icon set and choose an accent color. The icon is used everywhere the agent is shown, including the inbox, agents list, and channel cards.

## Create reusable skills at any time

Every Fleet agent has access to a built-in `generate-skill` skill. That means you can ask the agent to capture the current conversation, a one-off correction, or a new workflow as a reusable skill at any point, not only from the editor.

<Prompt description="Turn the conversation you just had into a reusable skill.">
Turn what we just did into a skill so you can repeat it next time.
</Prompt>

After the skill is created, share it with your workspace from the editor so other agents can use it. See [Skills](/langsmith/fleet/skills) for details on private vs. shared skills.

## Use a different model in each sub-agent

The main agent and each sub-agent can use a different model. This is useful when a sub-agent handles a narrow task that does not need a frontier model. Pointing it at a smaller, cheaper model can lower cost and latency without changing the main agent.

To set a sub-agent's model:

1. In the [LangSmith UI](https://smith.langchain.com), open the agent and click <Icon icon="pencil"/> **Edit Agent**.
2. Open the sub-agent in the **Sub-agents** section.
3. In the sub-agent's **Model** field, pick the model. If you do not set one, the sub-agent uses the main agent's model.

## Auto-improve agents from past conversations

Toggle on **Auto-improve** in the agent settings menu and the agent will run on a daily schedule, review the previous 24 hours of conversations, and update its own `AGENTS.md`, tool selection, or skills if it finds a clear improvement.

To enable it:

1. In the [LangSmith UI](https://smith.langchain.com), open the agent and click <Icon icon="pencil"/> **Edit Agent**.
2. Click the <Icon icon="settings"/> settings icon.
3. In the **Auto-improve** section, toggle the switch on.

<Note>
Auto-improve works by creating a daily [schedule](/langsmith/fleet/schedules) on the agent. The agent uses the `load_previous_conversations` tool to review recent activity before making any edits.
</Note>

## Inbox keyboard shortcuts

The [inbox](https://smith.langchain.com/agents/inbox) is keyboard-driven. Focus a thread row, then use:

| Shortcut | Action |
|----------|--------|
| `↑` / `↓` | Move between threads |
| `→` or `Enter` | Open the focused thread |
| `←` or `Esc` | Close the open thread or clear selection |
| `e` | Toggle read/unread |
| `x` | Add the thread to the bulk selection |
| `Backspace` / `Delete` | Delete the focused thread |

## Drag, drop, and paste files into chat

The chat input accepts files by drag-and-drop and by paste. Files are added as attachments to your next message. Up to 20 files can be attached at once and each file must be under 5 MB. Supported types include UTF-8 text files, images, audio, video, and PDFs.

You can also drag files out of the artifacts sidebar and drop them directly into the input to insert an `@`-mention referencing them.

## Import an agent from a ZIP

Drop a Fleet agent ZIP onto the **+ New Agent** page to bootstrap a new agent from its files. The importer reads the included tools, sub-agents, skills, and `AGENTS.md` and pre-fills the editor. This pairs with [**Download ZIP**](/langsmith/fleet/manage-agent-settings#download-agent-files) for moving an agent between workspaces.

## Inspect rendered HTML and send elements back to the agent

When the agent produces an HTML file, the chat renders it in an inspectable preview. Toggle inspect mode to click any element on the page and Fleet will send the element's tag and selector back to the agent in your next message. This is useful for asking the agent to revise a specific section of a generated page without copy-pasting selectors.

## Preview CSV files as a table

CSV files in the artifacts sidebar render as a paginated table instead of raw text. The card preview shows the first rows inline; opening the file fullscreen shows the whole table.

## Edit or reject email drafts before they are sent

When an email tool requires approval, the chat renders an editable email card with **To**, **Subject**, and **Body** fields. You can:

- **Send** the email as drafted.
- **Edit** the fields and then send the edited version.
- **Reject** the draft and provide a message that is fed back to the agent so it can try again.

This avoids re-prompting the agent in chat to make small wording changes.

## Answer clarifying questions inline

When an agent needs more information before continuing, it can ask a structured clarifying question with suggested answers. The chat renders a checkbox/radio UI so you can pick from the suggestions, or write a free-form answer, without retyping the question's context.

## Connect a custom model

Fleet supports any LLM API that follows the OpenAI chat completions spec or the Anthropic chat spec. This lets you point an agent at LiteLLM, Portkey, or any other proxy, or at a self-hosted model with a public endpoint. See [Custom models](/langsmith/fleet/essentials#custom-models) for setup.

## Pause and resume an agent without deleting it

Pausing an agent stops all of its [channels](/langsmith/fleet/channels) without losing configuration. Useful when you want to suspend an automated agent during an outage or while iterating on its instructions. See [Pause agent](/langsmith/fleet/manage-agent-settings#pause-agent).

## View agent traces from the editor

The settings menu has a **View agent traces** action that opens LangSmith filtered to runs from the current agent. This is the fastest path from a misbehaving conversation to the trace that produced it.

## Update instructions by chatting

You do not have to open the editor to change an agent's instructions. Tell the agent in chat to update them and it will edit `AGENTS.md` for you.

<Prompt description="Ask the agent to update its own instructions.">
Update your instructions to always respond in bullet points.
</Prompt>

See [Instructions](/langsmith/fleet/essentials#instructions) for more.
