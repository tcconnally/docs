The Headless Fleet API is the programmatic foundation for shipping production agents on LangSmith. Use it to OEM agents into your SaaS, build customer-facing agent products, or run internal tools on Fleet's runtime, while keeping control of your model, sandbox, memory, and identity.

## What you can build

- Customer-facing agent products embedded in your existing SaaS, with your branding and your auth.
- Internal tools and admin consoles that wrap Fleet for your own engineers and operators.
- Agent provisioning pipelines driven from infrastructure-as-code or CI.

## Why Headless Fleet

- **Bring your own model.** Use any model from any provider. No lock-in to a single vendor.
- **Bring your own sandbox.** Plug in Daytona, Modal, Runloop, LangSmith Sandboxes, or your own.
- **Open standards.** Agents are defined with [AGENTS.md](https://agents.md/) and [Agent Skills](https://agentskills.io/), and exposed over MCP, A2A, and Agent Protocol.
- **You own your memory.** Memory is stored in standard files and stays in your databases when you self-host LangSmith.

## Headless Fleet vs. Fleet

[Fleet](/langsmith/fleet) is the no-code product for building and running agents from a hosted UI. The Headless Fleet API exposes the same underlying platform as a set of REST endpoints. Agents, threads, skills, triggers, and integrations created through Fleet are available through the API, and resources created through the API appear in the Fleet UI.

## Authentication

Authentication is required for all Headless Fleet API requests. Include your LangSmith API key in the `{HEADER_NAME}` header on every request.

## Make your first request

The following request lists agents in your workspace:

```shell
curl --request GET \
  --url {BASE_URL}/v1/fleet/agents \
  --header '{HEADER_NAME}: LANGSMITH_API_KEY'
```

A successful response returns a paginated list:

```json
{
  "items": [
    { "id": "agt_...", "name": "My agent", "status": "active" }
  ],
  "next_cursor": null
}
```

Pass `next_cursor` as the `cursor` query parameter to fetch the next page. Use `page_size` to control the page size.
