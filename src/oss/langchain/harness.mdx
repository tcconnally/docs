An agent is a model plus a **harness**. The model reasons; the harness is the capability layer that defines what it can do.

For a simple task—answer a question, call one API—the harness is trivial: a model, a few tools, and a loop. But deep agents that run for minutes or hours, manage large amounts of context, coordinate specialized subagents, and need to stay reliable in production require more:

- **Tools**: functions, APIs, and databases the model can call
- **Execution environment**: a filesystem or sandbox where the agent reads, writes, and runs code
- **Context management**: memory, skills, and summarization that keep the agent coherent across long sessions
- **Delegation**: subagents that handle isolated subtasks in their own context windows, keeping the main agent's context clean
- **Steering**: human-in-the-loop approval at critical decision points

[`create_agent`][langchain.agents.create_agent] is the minimal harness—a model calling tools in a loop, with a middleware system that lets you extend it with exactly the capabilities you need. Every item above is opt-in.

The table below maps every built-in capability to its harness category. For the pre-assembled harness tuned for deep, long-running agents, see [Deep Agents](/oss/deepagents/harness).

| Category | What it covers |
|---|---|
| **[Execution environment](#execution-environment)** | Custom tools, virtual filesystem, code execution |
| **[Context management](#context-management)** | Summarization, memory, skills, prompt caching |
| **[Planning & delegation](#planning-and-delegation)** | Task tracking, subagents |
| **[Fault tolerance](#fault-tolerance)** | Retries, fallbacks, call limits |
| **[Guardrails](#guardrails)** | PII detection and redaction |
| **[Steering](#steering)** | Human-in-the-loop approval |

For the full middleware reference with configuration examples, see [Prebuilt middleware](/oss/langchain/middleware/built-in).

## Execution environment

The execution environment is how an agent acts in the world. It starts with the tools you wire up—custom functions, APIs, databases—and extends with middleware that adds a managed filesystem and code execution.

| Capability | How to add |
|---|---|
| Custom tools (functions, APIs) | `tools=` parameter on `create_agent` |
| Virtual filesystem + file tools | @[`FilesystemMiddleware`] |
| JavaScript REPL (QuickJS) | [`CodeInterpreterMiddleware`](/oss/deepagents/interpreters) (`langchain_quickjs`) |
| Persistent shell across turns | @[`ShellToolMiddleware`] |
| Isolated sandbox with shell execution | [`SandboxBackend`](/oss/deepagents/sandboxes) |

## Context management

Context management controls what the agent knows, how long it can run within token limits, and what it retains across sessions.

| Capability | How to add |
|---|---|
| History compression + large-result offloading | @[`SummarizationMiddleware`] |
| Agent-triggered summarization | @[`SummarizationToolMiddleware`] |
| Persistent instructions via `AGENTS.md` | @[`MemoryMiddleware`] |
| On-demand domain knowledge | @[`SkillsMiddleware`] |
| Prompt caching (Anthropic) | @[`AnthropicPromptCachingMiddleware`] |
| Dynamic tool filtering | @[`LLMToolSelectorMiddleware`] |

## Planning and delegation

Delegation lets agents break large problems into parallel or sequential units of work with isolated context windows.

| Capability | How to add |
|---|---|
| Todo list tracking across turns | @[`TodoListMiddleware`] |
| Sync subagent spawning | @[`SubAgentMiddleware`] |
| Background non-blocking subagents | @[`AsyncSubAgentMiddleware`] |

## Fault tolerance

Fault tolerance keeps agents running when models or tools fail transiently.

| Capability | How to add |
|---|---|
| Retry on model call failure | @[`ModelRetryMiddleware`] |
| Retry failed tool calls | @[`ToolRetryMiddleware`] |
| Fall back to an alternate model | @[`ModelFallbackMiddleware`] |
| Cap total model calls per session | @[`ModelCallLimitMiddleware`] |
| Cap total tool calls per session | @[`ToolCallLimitMiddleware`] |

## Guardrails

| Capability | How to add |
|---|---|
| Detect and redact PII from tool results | @[`PIIMiddleware`] |

## Steering

| Capability | How to add |
|---|---|
| Pause for human approval before tool calls | @[`HumanInTheLoopMiddleware`] |

## Composing from scratch

Every request through `create_agent` flows through a middleware stack. Each middleware hooks into one stage of the loop—before the model, after a tool call, around the full run—without forking the harness. Compose the stack your agent needs.

:::python

The following example builds a research assistant layer by layer. Stop wherever your use case is satisfied.

```python
from langchain.agents import create_agent

# The minimal harness: model + tools in a loop
agent = create_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=[search],
)
```

**Add execution environment**—give the agent a filesystem and the ability to run code:

```python
from langchain.agents import create_agent
from deepagents.backends import StateBackend
from deepagents.middleware import FilesystemMiddleware

backend = StateBackend()

agent = create_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=[search],
    middleware=[
        FilesystemMiddleware(backend=backend),
    ],
)
```

**Add context management**—compress history automatically and inject domain knowledge:

```python
from langchain.agents import create_agent
from deepagents.backends import StateBackend
from deepagents.middleware import FilesystemMiddleware, MemoryMiddleware, SkillsMiddleware, SummarizationMiddleware

backend = StateBackend()
model = "anthropic:claude-sonnet-4-6"

agent = create_agent(
    model=model,
    tools=[search],
    middleware=[
        FilesystemMiddleware(backend=backend),
        SummarizationMiddleware(model=model, backend=backend),
        MemoryMiddleware(backend=backend, sources=["./AGENTS.md"]),
        SkillsMiddleware(backend=backend, sources=["./skills/"]),
    ],
)
```

**Add planning and delegation**—track tasks and spawn subagents for isolated subtasks:

```python
from langchain.agents import create_agent
from langchain.agents.middleware import TodoListMiddleware
from deepagents import SubAgent
from deepagents.backends import StateBackend
from deepagents.middleware import FilesystemMiddleware, MemoryMiddleware, SkillsMiddleware, SubAgentMiddleware, SummarizationMiddleware

backend = StateBackend()
model = "anthropic:claude-sonnet-4-6"

researcher: SubAgent = {
    "name": "researcher",
    "description": "Deep-dives into a topic and returns a structured summary.",
    "system_prompt": "You are a research specialist. Search thoroughly and cite your sources.",
    "tools": [search],
}

agent = create_agent(
    model=model,
    tools=[search],
    middleware=[
        FilesystemMiddleware(backend=backend),
        SummarizationMiddleware(model=model, backend=backend),
        MemoryMiddleware(backend=backend, sources=["./AGENTS.md"]),
        SkillsMiddleware(backend=backend, sources=["./skills/"]),
        TodoListMiddleware(),
        SubAgentMiddleware(backend=backend, subagents=[researcher]),
    ],
)
```

**Add steering**—pause for human review before high-impact actions:

```python
from langchain.agents import create_agent
from langchain.agents.middleware import HumanInTheLoopMiddleware, TodoListMiddleware
from langgraph.checkpoint.memory import MemorySaver
from deepagents import SubAgent
from deepagents.backends import StateBackend
from deepagents.middleware import FilesystemMiddleware, MemoryMiddleware, SkillsMiddleware, SubAgentMiddleware, SummarizationMiddleware

backend = StateBackend()
model = "anthropic:claude-sonnet-4-6"

researcher: SubAgent = {
    "name": "researcher",
    "description": "Deep-dives into a topic and returns a structured summary.",
    "system_prompt": "You are a research specialist. Search thoroughly and cite your sources.",
    "tools": [search],
}

agent = create_agent(
    model=model,
    tools=[search],
    middleware=[
        FilesystemMiddleware(backend=backend),                                       # execution environment
        SummarizationMiddleware(model=model, backend=backend),                       # context management
        MemoryMiddleware(backend=backend, sources=["./AGENTS.md"]),                  # context management
        SkillsMiddleware(backend=backend, sources=["./skills/"]),                    # context management
        TodoListMiddleware(),                                                         # planning & delegation
        SubAgentMiddleware(backend=backend, subagents=[researcher]),                 # planning & delegation
        HumanInTheLoopMiddleware(interrupt_on={"write_file": True}),                # steering
    ],
    checkpointer=MemorySaver(),
)
```

:::

<Tip>
`create_deep_agent` pre-assembles this stack for long-running coding and research tasks—filesystem, summarization, subagents, and prompt caching are included by default. See [Deep Agents](/oss/deepagents/harness) for the full pre-built harness.
</Tip>
